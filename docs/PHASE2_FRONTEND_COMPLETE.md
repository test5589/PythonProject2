### 4. API 整合

#### 4.1 使用的 API 端點

**載入 K線**:
```javascript
GET /api/charts/candles?symbol=BTCUSDT&interval=60&limit=1000
```

**獲取交易對**:
```javascript
GET /api/charts/symbols
```

**獲取時間框架**:
```javascript
GET /api/charts/timeframes
```

**同步資料**:
```javascript
POST /api/sync/
{
  "symbol": "BTCUSDT",
  "interval": 60,
  "category": "crypto",
  "overwrite": false
}
```

#### 4.2 Vite 代理配置
```javascript
proxy: {
  '/api': {
    target: 'http://localhost:8001',
    changeOrigin: true,
  }
}
```

**好處**:
- 避免 CORS 問題
- 簡化前端請求路徑
- 開發環境友好

## 🔄 用戶流程

### 首次使用
```
1. 運行 start_all.bat
2. 等待後端和前端啟動
3. 瀏覽器自動打開（或手動訪問 localhost:5173）
4. 看到空圖表，提示"無K線資料，請先同步"
5. 點擊"同步資料"按鈕
6. 等待幾秒，資料同步完成
7. 圖表自動顯示K線
```

### 切換交易對
```
1. 點擊 Header 的交易對選擇器
2. 搜索或選擇想要的交易對（如 ETHUSDT）
3. 自動載入該交易對的K線
4. 如果無資料，點擊"同步資料"
```

### 切換時間框架
```
1. 點擊時間框架按鈕（如 "5m"）
2. 自動載入該時間框架的K線
3. 圖表立即更新顯示
```

### 過濾資料來源
```
1. 在控制面板選擇資料來源（All/Real/Aggregation）
2. 自動重新載入並過濾
3. 圖表只顯示選中的資料來源
```
