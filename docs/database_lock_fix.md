### 用戶報告的問題

```
1. 按下「暫時停止回補」按鈕

5. 之後「開始資料回補」按鈕一直無法點擊
```

### 影響
- ❌ 用戶無法暫停後重新開始回補
- ❌ 必須重啟GUI才能恢復
- ❌ 嚴重影響使用體驗

## 🔍 根本原因分析

### 問題1：鎖釋放邏輯錯誤

**舊代碼**：
```python
def run():
    try:
        # 嘗試獲取鎖
        if not _db_operation_lock.acquire(timeout=30):
            # 獲取失敗，記錄錯誤
            backfill_state_manager.set_error("資料庫鎖獲取超時")
            return  # 直接返回
    
    finally:
        # 🔴 問題：嘗試釋放未獲得的鎖！
        _db_operation_lock.release()
```

**問題**：
- 獲取鎖超時後直接 `return`
- `finally` 塊無論如何都會執行
- 嘗試釋放一個從未獲得的鎖 → 錯誤或鎖狀態混亂

---

### 問題2：固定等待時間不足

**舊代碼**：
```python
# 停止舊任務
backfill_state_manager.stop_backfill()
time.sleep(1.5)  # 🔴 固定等待1.5秒
# 繼續開始新任務
```

**問題**：
- 1.5秒可能不夠讓舊線程釋放鎖
- 舊線程可能還在執行批量插入
- 新任務嘗試獲取鎖 → 超時 → 失敗

---

### 問題3：按鈕狀態恢復

**理論上**：外層 `finally` 塊應該恢復按鈕

**實際**：
- 獲取鎖失敗後 `return`
- 外層 `finally` 應該執行
- **但是**：內層 `finally` 釋放鎖時可能出錯
- 導致按鈕狀態恢復失敗

---

## ✅ 解決方案

### 修復1：追蹤鎖的獲取狀態

```python
def run():
    lock_acquired = False  # 🟢 追蹤鎖是否成功獲取
    try:
        if not _db_operation_lock.acquire(blocking=False):
            if not _db_operation_lock.acquire(timeout=30):
                # 獲取失敗
                return
        
        lock_acquired = True  # 🟢 標記鎖已獲取
        
        # ... 執行回補 ...
    
    finally:
        # 🟢 只在成功獲取鎖時才釋放
        if lock_acquired:
            _db_operation_lock.release()
            gui_safe_log("🔓 資料庫操作鎖已釋放")
        else:
            gui_safe_log("⚠️ 未獲取鎖，無需釋放")
```

**好處**：
- ✅ 不會嘗試釋放未獲得的鎖
- ✅ 避免鎖狀態混亂
- ✅ 清楚知道鎖的狀態

---

### 修復2：主動等待鎖釋放

```python
# 停止舊任務
backfill_state_manager.stop_backfill()

# 🟢 主動等待資料庫鎖被釋放（最多10秒）
wait_start = time.time()
max_wait = 10

while time.time() - wait_start < max_wait:
    # 嘗試獲取鎖（非阻塞）
    if _db_operation_lock.acquire(blocking=False):
        # 成功獲取，立即釋放（我們只是測試）
        _db_operation_lock.release()
        gui.log("✅ 舊任務已停止，資料庫鎖已釋放")
        break
    # 沒獲取到，短暫等待後重試
    time.sleep(0.5)
else:
    # 超時了
    gui.log("⚠️ 警告：舊任務可能未完全停止（超時10秒）")
    gui.log("💡 如果下一步出現鎖超時，請重啟GUI")
```

**好處**：
- ✅ 主動檢查鎖是否可用
- ✅ 最多等10秒（而不是固定1.5秒）
- ✅ 通常2-5秒就能檢測到鎖釋放
- ✅ 即使超時也有清楚提示

---

### 修復3：更好的錯誤訊息

```python
if not _db_operation_lock.acquire(timeout=30):
    gui_safe_log("❌ 無法獲取資料庫鎖，操作超時（30秒）")
    gui_safe_log("💡 可能原因：暫停的任務未完全停止，或其他操作正在進行")
    gui_safe_log("💡 建議：稍後再試，或重啟GUI")
    backfill_state_manager.set_error("資料庫鎖獲取超時")
    return
```

**好處**：
- ✅ 解釋為什麼失敗
- ✅ 提供可能的原因
- ✅ 建議解決方案
