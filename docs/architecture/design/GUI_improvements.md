## 🎯 功能說明

### **暫停/恢復功能**
1. **暫停回補**：
   - 點擊「⏸️ 暫時停止回補」
   - 回補進程暫停，記錄當前位置
   - 按鈕文字變為「▶️ 回補資料重新開始」

2. **恢復回補**：
   - 點擊「▶️ 回補資料重新開始」
   - 從暫停位置繼續回補
   - 按鈕文字變回「⏸️ 暫時停止回補」

### **完全停止功能**
1. 點擊「⏹️ 完全停止回補」
2. 彈出確認對話框
3. 確認後立即停止回補，已處理的資料保存，未處理的放棄

### **Real 資料覆蓋確認**
1. **觸發時機**：回補資料時發現該時間點已有 real 資料
2. **對話框內容**：
   - 顯示衝突資料的詳細資訊（交易對、間隔、時間）
   - 提供四個選項讓用戶選擇
3. **選項說明**：
   - **只覆蓋這一筆**：僅覆蓋當前這筆，下次遇到再詢問
   - **覆蓋所有**：本次會話內所有衝突都自動覆蓋
   - **跳過這一筆**：僅跳過當前這筆，下次遇到再詢問
   - **跳過所有**：本次會話內所有衝突都自動跳過

## 🔒 資料優先級規則

1. **real** > **interpolated** > **Aggregation** > **inferior-Aggregation**
2. **real vs real**：需要用戶確認是否覆蓋
3. **其他情況**：優先級高的資料不會被低優先級資料覆蓋

## 📝 使用範例

### **情境 1：正常回補資料**
1. 設定時間範圍：2025-11-01 00:00:00 ~ 2025-11-01 23:59:59
2. 選擇回補間隔：5分
3. 點擊「🚀 開始回補資料」
4. 系統開始抓取 5 分鐘級別資料

### **情境 2：暫停後繼續**
1. 回補進行中，需要暫停
2. 點擊「⏸️ 暫時停止回補」
3. 進程暫停
4. 準備好後點擊「▶️ 回補資料重新開始」
5. 從暫停位置繼續

### **情境 3：發現 Real 資料衝突**
1. 回補過程中發現已存在 real 資料
2. 彈出確認對話框
3. 用戶選擇「覆蓋所有（不再詢問）」
4. 後續所有衝突自動覆蓋，不再詢問
5. 重啟 GUI 後，此設定重置

## ⚠️ 注意事項

1. **間隔選單**：
   - 只影響「回補資料」功能
   - 不影響「抓取最新資料」（固定 1 分鐘）
   - 不影響「啟動 1秒監控」（固定 1 秒）

2. **資料完整性**：
   - 所有抓取的資料都會驗證 11 個必要欄位
   - 資料不完整時會停止並報錯

3. **覆蓋設定**：
   - 覆蓋設定僅在本次 GUI 會話有效
   - 關閉 GUI 後設定會遺失
   - 下次啟動 GUI 會重新詢問

4. **暫停記錄**：
   - 暫停位置僅在記憶體中保存
   - 關閉 GUI 後暫停記錄會遺失
