## ✅ 解決方案

### 1. 精簡日誌過濾邏輯修復

#### 問題原因分析
```python
# 舊邏輯（有問題）
if self.compact_var.get():  # 只在精簡模式下過濾
    if "筆跳過" in msg and "最近" in msg:
        return
```

**問題**:
- 聚合訊息總是被產生（在 backfill_data.py）
- 只有在精簡模式下才過濾
- 用戶不打勾時，仍然看到聚合訊息
- 但用戶期望：不打勾時應該看詳細訊息，不是聚合訊息

#### 新邏輯（已修復）
```python
# 新邏輯（正確）
is_compact = self.compact_var.get()

# 識別聚合訊息
is_aggregated_skip = "筆跳過" in msg and "最近" in msg and "•" in msg
is_aggregated_success = "筆 •" in msg and "最近" in msg and "批次" in msg
is_aggregated_msg = is_aggregated_skip or is_aggregated_success

if is_compact:
    # 精簡模式：過濾詳細進度，保留聚合訊息
    if "(總計:" in msg and "批次" in msg:
        return  # 過濾詳細進度
else:
    # 非精簡模式：過濾聚合訊息，保留詳細訊息
    if is_aggregated_msg:
        return  # 過濾聚合訊息 ← 修復點！
```

**修復效果**:
| 模式 | 顯示內容 | 效果 |
|------|----------|------|
| **不打勾（非精簡）** | 顯示每筆詳細資料 | ✅ 符合預期 |
| **打勾（精簡）** | 顯示聚合摘要 | ✅ 符合預期 |

### 2. 進度條UI實現（階段2優化）

#### 設計目標
1. ✅ 清晰的視覺進度顯示
2. ✅ 減少日誌窗口的負擔
3. ✅ 保留異常情況的詳細記錄（最近5-10筆）
4. ✅ 實時統計信息（插入/跳過/速度）

#### 組件架構

```
BackfillProgressBar (UI組件)
├── 主進度條（總體進度）
│   ├── 當前貨幣對@間隔
│   ├── 進度百分比
│   └── 當前/總數
│
├── 批次進度條（當前批次）
│   ├── 批次編號
│   └── 批次內進度
│
├── 統計信息
│   ├── 左側：插入數/跳過數
│   └── 右側：已用時間/速度
│
└── 異常記錄區（最近10筆）
    ├── 跳過訊息 ⏭️
    ├── 錯誤訊息 ❌
    └── 警告訊息 ⚠️

ProgressBarLogger (邏輯處理)
├── 解析日誌訊息
├── 更新進度條
├── 記錄異常情況
└── 計算統計數據
```

#### 實現細節

##### BackfillProgressBar（UI組件）

**主要功能**:
```python
class BackfillProgressBar:
    def __init__(self, parent_frame):
        # 主進度條
        self.main_progress = ttk.Progressbar(...)
        
        # 批次進度條
        self.batch_progress = ttk.Progressbar(...)
        
        # 統計信息
        self.left_stats = ttk.Label(...)   # 插入/跳過
        self.right_stats = ttk.Label(...)  # 時間/速度
        
        # 異常記錄區（黃色背景）
        self.exception_text = tk.Text(height=5, bg="#FFF3CD")
```

**關鍵方法**:
- `update_main_progress()`: 更新總體進度
- `update_batch_progress()`: 更新批次進度
- `update_stats()`: 更新統計信息
- `add_exception()`: 添加異常記錄（最多10筆）
- `set_complete()`: 設置完成狀態

##### ProgressBarLogger（日誌處理器）

**主要功能**:
```python
class ProgressBarLogger:
    def log(self, msg: str):
        # 使用正則表達式解析日誌
        
        # 跳過訊息 → 記錄到異常區
        if "筆跳過" in msg:
            self.progress_bar.add_exception('skip', msg)
            # 解析總跳過數
        
        # 錯誤訊息 → 記錄到異常區
        elif "❌" in msg or "錯誤" in msg:
            self.progress_bar.add_exception('error', msg)
        
        # 批次訊息 → 更新批次進度
        elif "📦 批次" in msg:
            # 解析批次編號
            self.progress_bar.update_batch_progress(...)
        
        # 進度訊息 → 更新主進度
        elif "總:" in msg:
            # 解析總進度
            self.progress_bar.update_main_progress(...)
```
